一、功能概述与核心突破
================================================================================

本工具用于实现湖南省家庭医生签约的全流程自动化，整合了健康卡平台和公卫
3.0系统两套平台的操作，实现"一键签约"。

v3.0 版本实现了以下核心技术突破:

  【突破1】人脸认证绕过
    通过 updateRpc 接口，直接将健康卡的人脸认证状态设置为"已验证"，
    无需真实人脸扫描。所有年龄段均有效。

  【突破2】自动创建居民申请合同
    通过 insertJtysqy 接口，直接在健康卡平台创建居民申请合同
    (STATUS=6)。无需从3.0系统医生端发起签约申请。

  【突破3】自动确认签约
    通过 editqr 接口，将待确认合同(STATUS=5或6)自动确认为已签约
    (STATUS=1)。

  【突破4】3.0档案修改（年龄绕行）
    对18-60岁居民，可临时修改3.0系统档案中的出生日期，使系统判定为
    <18岁，从而免除绑卡时的人脸验证要求，完成后自动恢复原信息。

核心功能一览:
  • 一键全流程签约：绕过人脸 → 查询状态 → 创建合同 → 确认签约
  • 支持全人群：未签约、医生申请(状态5)、居民申请(状态6)均可处理
  • 批量操作：自动逐个处理选中的健康卡，实时显示进度
  • 3.0系统联动：可同步机构代码、签约团队、服务包等配置
  • OpenID 自动抓取：内置代理服务器，一键抓取微信 OpenID

二、安装与启动
================================================================================

使用前提:
  • 操作系统：Windows 7 / Windows 10 / Windows 11
  • 网络环境：电脑能正常访问互联网
  • 显示器分辨率：建议 1280×720 及以上

安装步骤:
  1. 获取 GulfSign.exe 文件
  2. 将 GulfSign.exe 复制到电脑上任意文件夹中
     （如：桌面、D:\工具 等均可）
  3. 双击 GulfSign.exe 即可启动程序

  无需安装任何额外软件，双击即用。

首次启动说明:
  • 首次启动时，Windows 可能弹出安全提示"Windows已保护你的电脑"，
    点击"更多信息"，然后点击"仍要运行"即可。
  • 如果杀毒软件拦截，请将 GulfSign.exe 添加到杀毒软件的信任列表中。
  • 程序启动后会在同目录下自动生成一个配置文件
    （gulfsign_config.json），用于保存您的使用偏好，请勿删除。

4.1 前置准备：获取 OpenID
────────────────────────────────────────

  OpenID 是微信用户在健康卡平台的唯一标识，获取一次即可长期使用。

  1. 打开湾流签约助手，点击第三个标签页「获取OpenID」

  2. 点击「▶ 启动代理」
     系统自动完成:
       ✓ 生成并安装 CA 证书（首次会弹出确认框，点击「是」）
       ✓ 设置系统代理

  3. 打开电脑版微信 → 搜索小程序「我的健康卡」→ 点击进入

  4. 应用自动抓取并显示 OpenID
     日志显示: [ok] 发现 OpenID: oxVqWx...

  5. 点击列表中的 OpenID → 点击「★ 使用选中的OpenID」
     应用自动切换到「健康卡签约」页面并填入

  6. 点击「⏹ 停止代理」（重要：不停止会影响正常上网）

  此后每次使用只需填入之前获取的 OpenID 即可，无需重复抓取。


────────────────────────────────────────
4.2 前置准备：微信绑定签约对象的健康卡
────────────────────────────────────────

  在微信中将需要签约的居民健康卡绑定到您的账号下。
  每个 OpenID 最多同时绑定 9 张卡。

  操作步骤:
  1. 打开微信 → 搜索小程序「我的健康卡」→ 进入

  2. 点击「家庭管理」或「添加家庭成员」

  3. 输入签约对象的 姓名 和 身份证号 完成绑定
     • <18岁 或 >60岁：直接绑定，无需人脸验证
     • 18-60岁：需要本人人脸验证
       （后续版本将通过年龄绕行机制优化此步骤）

  4. 重复操作，每批最多绑定 9 张卡

  绑卡方式:
  • 「我的健康卡」微信小程序（推荐）
  • 「湖南省居民健康卡」微信公众号
  • 「湘易办」APP → 省卫健委


────────────────────────────────────────
4.3 连接健康卡平台
────────────────────────────────────────

  1. 点击标签页2「健康卡签约」

  2. 在「微信OpenID」栏填入 OpenID
     （如已通过「获取OpenID」页面抓取，会自动填入）

  3. 点击「连接」
     成功后状态栏显示: Token获取成功

  4. 点击「刷新卡列表」
     表格中显示该 OpenID 下绑定的所有健康卡信息:

     列名说明:
     • # — 序号
     • 姓名 — 持卡人姓名
     • 身份证号 — 持卡人身份证（部分位数可能被遮掩）
     • 年龄 — 从身份证号计算得出
     • 人群分类 — 老年人 / 儿童 / 普通 等
     • 性别 — 男 / 女
     • 人脸认证 — 当前人脸认证状态（已认证 / 未认证）
     • 需绕行 — 是否需要年龄绕行（18-60岁显示"是"）
     • 关系 — 与 OpenID 持有者的关系


────────────────────────────────────────
4.4 签约配置（自动/手动）
────────────────────────────────────────

  连接成功后，需要配置签约参数。有两种方式:

  方式一：从3.0系统自动同步（推荐）
  ────────────────────────────────
  1. 先在标签页1「3.0系统签约」中登录3.0系统
  2. 回到标签页2，点击「从3.0系统同步配置」
  3. 系统自动获取并填入:
     • 机构代码
     • 签约医生
     • 签约团队
     • 服务包信息

  方式二：手动填写
  ─────────────────
  直接在签约配置区填写以下信息:
  • 机构代码 — 您所在机构的代码（如 43072600XX）
  • 签约团队 — 签约团队名称
  • 签约医生 — 签约医生姓名
  • 协议开始 — 签约协议开始日期（如 20260101，留空=今天）
  • 协议结束 — 签约协议结束日期（如 20291231，留空=自动3年）

  配置选项:
  • ☑ 自动创建合同(未签约居民)
    勾选后，对于尚未签约的居民，软件会自动创建居民申请合同并确认。
    不勾选则只处理已有待确认合同的居民。
    建议：保持勾选（默认已勾选）。

  提示：所有配置信息会自动保存，下次启动自动恢复。


────────────────────────────────────────
4.5 一键全流程签约
────────────────────────────────────────

  配置完成后，即可执行签约:

  1. 在卡列表中选择要签约的卡
     • 默认全部选中（蓝色背景）
     • 点击某行可取消/重新选中
     • 勾选/取消「全选」控制全部

  2. 点击「▶ 一键全流程签约」

  3. 弹出确认对话框，点击「是」开始

  4. 软件自动逐个处理每张卡，每张卡的处理流程:

     步骤1  设置人脸认证绕过 (updateRpc → rpc=1)
         ↓
     步骤2  查询签约状态 (querybyidcardqyjg)
         ↓
     步骤3  根据状态自动处理:
            • 已签约 → 跳过
            • 有待确认合同 → 直接确认
            • 未签约 → 创建合同 → 确认
         ↓
     步骤4  确认签约 (editqr → STATUS=1)

  5. 实时查看进度:
     • 进度条显示已处理 / 总数
     • 右侧显示成功数和失败数
     • 日志区逐条记录每步操作结果

  6. 日志输出示例:

     [09:15:01] 处理 [1/5] 张三 (430726********1234)
     [09:15:01]   设置人脸认证绕过: 已完成实名认证。
     [09:15:02]   签约状态: 未签约
     [09:15:02]   未签约，将创建居民申请合同
     [09:15:02]   签约团队: 全科家医团队
     [09:15:02]   服务包: 基本公共卫生服务包,基本医疗服务包
     [09:15:03]   ✓ 居民申请合同已创建 (STATUS=6)
     [09:15:03]   确认合同: a1b2c3d4e5f6...
     [09:15:04]   ✓ 签约确认成功! (2.1s)
     ─────────────────────────────────────────
     [09:15:04] 处理 [2/5] 李四 (430726********5678)
     [09:15:04]   设置人脸认证绕过: 已完成实名认证。
     [09:15:05]   签约状态: 已签约
     [09:15:05]   已签约，跳过

  7. 表格中行背景色变化:
     • 绿色 — 签约成功
     • 红色 — 签约失败
     • 黄色 — 跳过（已签约或其他原因）

  8. 全部处理完成后，日志显示汇总:
     签约完成! 成功: 4, 失败: 0, 跳过: 1, 耗时: 10.2s

  随时可点击「⏹ 停止」终止操作（已完成的签约不会被撤销）。


────────────────────────────────────────
4.6 解绑已签约的卡 → 绑定下一批
────────────────────────────────────────

  签约完成后，解绑已处理的卡，腾出名额给下一批:

  1. 打开微信 → 「我的健康卡」小程序
  2. 进入「家庭管理」→ 点击已签约的卡
  3. 选择「解绑」或「移除家庭成员」
  4. 解绑完成后，绑定下一批签约对象（回到 4.2）
  5. 重复循环，直到全部处理完成

  循环流程图:

    ┌──→ ① 微信绑定9张卡（手动）
    │          ↓
    │    ② 软件「刷新卡列表」
    │          ↓
    │    ③ 点击「一键全流程签约」（全自动, 约10-20秒/批）
    │          ↓
    │    ④ 微信解绑已签约的卡（手动）
    │          ↓
    └── 还有未处理的？─→ 是 → 继续循环
                        → 否 → 全部完成!

五、自动签约引擎工作原理
================================================================================

────────────────────────────────────────
5.1 全自动流程详解
────────────────────────────────────────

  软件内置的签约引擎 (SigningEngine) 协调两个平台完成签约:

  平台1: 健康卡医疗机构联网平台 (jkkyljl.hnhfpc.gov.cn)
    → 获取Token、卡列表、设置人脸认证、创建合同、确认签约

  平台2: 公卫3.0系统 (ggws.hnhfpc.gov.cn)
    → 同步签约配置（团队、服务包）、档案修改（年龄绕行）

  自动化流程:

  ┌─────────────────────────────────────────────────────────────────┐
  │                                                                 │
  │  getToken        获取JWT认证令牌（有效期约30天）                 │
  │       ↓                                                         │
  │  newlist          获取该OpenID下绑定的所有健康卡                 │
  │       ↓                                                         │
  │  updateRpc        将健康卡设为"已认证" (rpc=1)                   │
  │       ↓            无需真实人脸，直接修改认证状态                │
  │  querybyidcard    查询居民签约状态和GUID                        │
  │       ↓                                                         │
  │  insertJtysqy     创建居民申请合同 (STATUS=6)                   │
  │       ↓            仅在居民未签约且启用"自动创建"时执行          │
  │  editqr           确认签约 (STATUS → 1)                         │
  │                    支持确认STATUS=5和STATUS=6的合同              │
  │                                                                 │
  └─────────────────────────────────────────────────────────────────┘


────────────────────────────────────────
5.2 各种签约状态的自动处理
────────────────────────────────────────

  软件会自动判断每位居民的当前签约状态，选择最优操作:

  ┌─────────────────────────────────────────────────────────────────┐
  │  居民当前状态        软件自动执行的操作        最终结果          │
  ├─────────────────────────────────────────────────────────────────┤
  │                                                                 │
  │  未签约          →  创建居民申请合同         → 已签约 ★        │
  │  (STATUS=1)         + 自动确认                  (STATUS=1)      │
  │                     (全自动，无需人工干预)                       │
  │                                                                 │
  │  医生申请        →  直接确认签约             → 已签约 ★        │
  │  (STATUS=5)         (editqr确认)                (STATUS=1)      │
  │                                                                 │
  │  居民申请        →  直接确认签约             → 已签约 ★        │
  │  (STATUS=6)         (editqr确认)                (STATUS=1)      │
  │                                                                 │
  │  已签约          →  跳过（无需处理）         → 保持不变        │
  │  (STATUS=0)                                                     │
  │                                                                 │
  └─────────────────────────────────────────────────────────────────┘

  关键优势:
  • 未签约居民无需先从3.0系统发起医生申请，软件直接创建+确认
  • 已有待确认合同（医生申请或居民申请）的，直接确认
  • 已签约的自动跳过，不会重复操作


────────────────────────────────────────
5.3 年龄绕行机制（18-60岁人脸认证绕过）
────────────────────────────────────────

  背景: 健康卡平台对18-60岁居民的绑卡操作要求人脸验证，
  而<18岁和>60岁的居民可以直接绑卡（免人脸）。

  绕行原理:
  健康卡平台通过身份证号中的出生日期计算年龄。如果临时修改
  3.0系统档案中的身份证号（改变出生年份使其<18岁），则该
  居民在绑卡时可免除人脸验证。绑卡完成后再恢复原始信息。

  在卡列表的「需绕行」列中:
  • 显示"是" → 该居民为18-60岁，绑卡时需要人脸验证
  • 空白     → 该居民为<18岁或>60岁，绑卡免人脸

  注意: 年龄绕行解决的是绑卡环节的人脸验证问题。
  签约确认环节的人脸验证已通过 updateRpc 完全绕过，不受年龄限制。

六、3.0系统辅助签约（标签页一）
================================================================================

  标签页1「3.0系统签约」可用于两个目的:

  目的1: 同步签约配置到健康卡签约页面
  ────────────────────────────────────
  1. 输入3.0系统的账号和密码，点击「登录」
  2. 登录成功后，切换到标签页2
  3. 点击「从3.0系统同步配置」
  4. 自动获取机构代码、签约医生、签约团队、服务包等信息

  目的2: 批量发起医生端签约申请（可选）
  ──────────────────────────────────────
  如果更希望走"医生申请 → 健康卡确认"的传统流程:
  1. 登录3.0系统
  2. 查询未签约居民
  3. 选中后点击「开始签约」批量发起（变为STATUS=5医生申请）
  4. 再到标签页2通过健康卡平台确认签约

  注意: v3.0版本中，标签页2的全流程签约已支持直接创建合同+确认，
  通常无需先在3.0系统发起医生申请，大幅简化了操作步骤。

七、配置自动保存说明
================================================================================

程序会自动记住您的使用偏好，下次启动时自动恢复:

  自动保存的内容:
  • 系统地址、登录账号（密码不保存，每次需重新输入）
  • 机构代码、签约医生、签约团队
  • 微信 OpenID
  • 健康卡签约配置（机构代码、团队、医生、协议日期）
  • 间隔时间设置

  保存时机:
  • 每次登录成功后自动保存
  • 每次开始签约时自动保存
  • 同步3.0配置后自动保存
  • 关闭程序时自动保存

  配置文件:
  • 保存在 GulfSign.exe 同目录下的 gulfsign_config.json 文件中
  • 如需重置所有配置，删除该文件即可恢复默认设置

八、批量操作最佳实践
================================================================================

  适合需要大量签约（数百~数千人）的场景:

  1. 准备工作
     • 获取 OpenID（一次性操作）
     • 登录3.0系统 → 同步签约配置
     • 准备待签约居民的姓名和身份证号列表

  2. 高效操作节奏
     • 每轮绑定9张卡 → 一键签约（约15-30秒）→ 解绑 → 下一轮
     • 每轮处理时间约2-3分钟（含手动绑/解绑）
     • 每小时可处理约 180-270 人

  3. 多微信号并行
     • 准备多个微信号，每个获取一次 OpenID
     • 多个窗口同时操作，效率翻倍
     • 每个微信号绑9张卡，互不影响

  4. 首次使用建议
     • 先绑定2-3张卡测试流程
     • 确认签约成功后再大批量操作
     • 注意查看日志中的错误信息

  5. 解绑注意事项
     • 微信对健康卡解绑有7天冷却期限制
     • 可通过「湘易办」APP → 省卫健委 解绑（可能无此限制）
     • 建议准备多个微信号轮换使用，避免受冷却期影响

  6. 签约量限制
     • 每个3.0系统医生ID通常有每日签约限额（约80人次/天）
     • 此限制针对3.0系统发起的医生申请
     • 通过健康卡平台直接创建居民申请（insertJtysqy）可能不受此限制
     • 如遇限制，可更换医生账号继续操作

九、常见问题与解决方案
================================================================================

问：点击「连接」后提示失败
答：请检查 OpenID 是否正确填写。OpenID 通常以"oxVqWx"开头。
    如果 OpenID 以"oK-UHu"开头，需要重新抓取正确的 OpenID。

问：「刷新卡列表」后表格为空
答：该 OpenID 下没有绑定任何健康卡。请先在微信中绑定。

问：「一键全流程签约」提示"签约团队GUID为空"
答：签约配置不完整。解决方法:
    (1) 先在标签页1登录3.0系统
    (2) 回到标签页2，点击「从3.0系统同步配置」
    (3) 确认机构代码、签约团队已自动填入

问：「一键全流程签约」提示"服务包GUID为空"
答：与上一个问题相同，需要先同步3.0系统配置以获取服务包信息。

问：日志显示"无法获取居民GUID (该卡可能未在平台注册)"
答：该健康卡在健康卡平台上没有对应的居民记录。
    可能是卡注册不完整，需要在微信中重新绑定该卡。

问：日志显示"已签约，跳过"
答：该居民已有有效签约合同，无需重复签约。属于正常情况。

问：日志显示"人脸认证设置失败"
答：可能是 Token 已过期。点击「连接」重新获取 Token 后重试。

问：卡列表中「需绕行」列显示"是"是什么意思？
答：该居民为18-60岁，在微信绑卡时需要本人人脸验证。
    这不影响签约确认流程（已通过 updateRpc 绕过），
    只影响前期的绑卡操作。

问：「从3.0系统同步配置」提示"请先在3.0系统签约页面登录"
答：需要先切换到标签页1「3.0系统签约」，输入账号密码完成登录，
    然后再回到标签页2点击同步。

问：解绑时提示"7天内不允许解绑"
答：微信对健康卡解绑有风控限制。建议:
    (1) 准备多个微信号轮换使用
    (2) 尝试通过「湘易办」APP 解绑
    (3) 合理安排绑定节奏

问：可以同时打开多个程序窗口吗？
答：可以。使用不同的 OpenID 操作不同的健康卡批次，互不影响。

问：首次启动弹出安全警告
答：Windows 对未签名程序的正常提示。
    点击"更多信息" → "仍要运行"即可。

问：签约完成后如何验证结果？
答：(1) 在3.0系统中查询该居民，确认签约状态为"已签约"
    (2) 再次「刷新卡列表」并签约，已签约的卡会显示"已签约，跳过"
