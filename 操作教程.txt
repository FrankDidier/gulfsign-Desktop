================================================================================
一、功能概述
================================================================================

本工具用于在湖南省基层卫生信息系统（公卫3.0）中实现家庭医生签约的自动化
批量操作。

核心功能：
  • 自动登录公卫3.0系统
  • 按多种条件查询居民列表（签约状态、协议到期日、姓名、身份证号等）
  • 支持分页查询（首页）和全量查询（加载所有页面数据）
  • 逐行选中/取消选中居民，或一键全选
  • 批量自动签约（自动发起 + 自动确认，支持暂停、继续、停止）
  • 签约进度实时显示（进度条、成功/失败计数、速度统计）
  • 签约结果按颜色标识（绿色=成功，红色=失败）
  • 运行日志实时记录每一步操作结果
  • 居民列表导出为CSV文件（可用Excel直接打开）


================================================================================
二、安装与启动
================================================================================

使用前提：
  • 操作系统：Windows 7 / Windows 10 / Windows 11
  • 网络环境：电脑能正常访问 https://ggws.hnhfpc.gov.cn
  • 显示器分辨率：建议 1280×720 及以上

安装步骤：
  1. 获取 GulfSign.exe 文件
  2. 将 GulfSign.exe 复制到电脑上任意文件夹中
     （如：桌面、D:\工具 等均可）
  3. 双击 GulfSign.exe 即可启动程序

  无需安装任何额外软件，双击即用。

首次启动说明：
  • 首次启动时，Windows 可能弹出安全提示"Windows已保护你的电脑"，
    点击"更多信息"，然后点击"仍要运行"即可。
  • 如果杀毒软件拦截，请将 GulfSign.exe 添加到杀毒软件的信任列表中。
  • 程序启动后会在同目录下自动生成一个配置文件
    （gulfsign_config.json），用于保存您的使用偏好，请勿删除。

四、详细操作步骤
================================================================================

────────────────────────────────────────
4.1 登录系统
────────────────────────────────────────

  1. 启动程序后，"系统地址"栏默认已填入公卫3.0的地址：
     https://ggws.hnhfpc.gov.cn
     通常无需修改。如果贵单位使用不同的服务器地址，请替换。

  2. 在"账号"栏输入3.0系统的登录账号。

  3. 在"密码"栏输入密码。密码会以星号 (***) 显示，保护隐私。

  4. 点击【登 录】按钮。

  5. 登录过程中：
     • 状态栏会显示蓝色文字"正在登录..."
     • 底部日志区会记录登录进度

  6. 登录成功后：
     • 状态栏变为绿色，显示"登录成功: 账号名 (机构名称)"
     • "机构代码"栏会自动填入当前账号所属机构的代码
     • "签约医生"栏会自动填入当前登录医生的姓名
     • "签约团队"栏会自动填入默认的签约团队名称
     • 这些信息均从系统自动提取，无需手动填写

  7. 登录失败时：
     • 状态栏变为红色，显示具体的错误原因
     • 请检查账号密码是否正确，网络是否正常

  注意：上一次登录使用的账号、机构代码、医生名等信息会自动保存，
        下次启动时自动恢复（密码出于安全考虑不会保存）。

────────────────────────────────────────
4.2 查询居民列表
────────────────────────────────────────

  登录成功后即可查询居民数据。

  基本查询：
  1. 在"签约状态"下拉框中选择要查询的状态：
     • 未签约  — 尚未签约的居民（最常用）
     • 已签约  — 已经有有效签约的居民
     • 医生申请 — 医生端已发起签约但尚未确认的居民
     • 居民申请 — 居民端已发起签约但医生端未确认的居民
     • 拒绝签约 — 被拒绝的签约记录
     • 全部    — 不限状态，查询所有居民

  2. "机构代码"通常在登录时已自动填入，无需手动修改。

  3. 点击【查询(首页)】加载第一页数据（通常为100条）。
     或点击【查询全部】加载该条件下的全部居民数据。

  【查询(首页)】与【查询全部】的区别：
     • 查询(首页)：速度快，只加载第一页（约100条记录）。
       适合快速预览，或确认查询条件是否正确。
     • 查询全部：加载所有页面的数据。当居民数量较多时（如数百、
       数千人），需要一定加载时间。加载过程中会显示进度
       （如"已加载 200 / 1500 ..."）。
       批量签约前建议使用此选项，确保所有目标居民都已加载。

  高级筛选：
  在查询条件的第二行，可使用以下筛选项缩小范围：

  • 协议到期日(前)：输入日期（格式如 20260404），查询协议到期日
    早于该日期的居民。需配合"已签约"状态使用，用于找出协议已过期
    但未解约的居民。留空则不限制。

  • 姓名：输入居民姓名进行查询。

  • 身份证：输入身份证号查询特定居民。

  查询完成后：
  • 表格中会显示所有查询结果
  • 状态栏显示"共 XX 条记录"
  • 默认所有居民都处于选中状态（蓝色高亮背景）

────────────────────────────────────────
4.3 筛选与选择居民
────────────────────────────────────────

  查询结果加载后，可对要签约的居民进行精确选择：

  单个选中/取消选中：
  • 直接点击表格中的某一行，该行会在"选中"（蓝色背景）和
    "未选中"（无背景）之间切换。

  全选/全不选：
  • 勾选表格上方的"全选"复选框 → 所有居民被选中
  • 取消勾选"全选"复选框 → 所有居民取消选中

  选中计数：
  • 表格上方始终显示"已选: X / Y"，其中 X 是当前选中数量，
    Y 是总记录数。

  只有被选中（蓝色背景）的居民才会在批量签约时被处理。

────────────────────────────────────────
4.4 批量签约
────────────────────────────────────────

  确认好要签约的居民后，开始批量签约：

  签约前准备：
  1. 确认"签约医生"栏显示正确的医生姓名。
     （登录后一般会自动填入，如需更换可手动修改）

  2. 确认"签约团队"栏显示正确的团队名称。
     （登录后一般会自动填入，如需更换可手动修改）

  3. 设置"间隔(秒)"：每签约一人后的等待时间。
     • 建议值：0.3 ~ 1.0 秒
     • 设置过快（如 0）可能导致服务器限流或拒绝请求
     • 设置过慢则效率降低
     • 默认 0.5 秒，一般无需修改

  开始签约：
  4. 点击【▶ 开始签约】按钮。

  5. 程序会弹出确认对话框："即将对 X 位居民执行自动签约，是否继续？"
     点击"是"开始，点击"否"取消。

  签约过程中：
  6. 进度条会实时更新，显示已处理数量和总数（如 12 / 50）。

  7. 右侧显示实时统计信息：成功数、失败数、平均速度。

  8. 日志区会逐条记录每位居民的签约结果，含三种结果类型：
     • ✓ 张三 已签约 (1.2秒)          — 绿色文字，签约完全成功
     • ◎ 李四 已发起签约 [待健康卡确认]  — 橙色文字，已发起但需外部确认
     • ✗ 王五 失败: 错误信息            — 红色文字，签约失败

  9. 表格中对应居民行的背景色会随之改变：
     • 绿色背景 — 签约成功或已发起
     • 红色背景 — 签约失败

  暂停/继续：
  10. 签约过程中，随时可点击【⏸ 暂停】暂停操作。
      暂停后按钮变为【▶ 继续】，点击即可恢复签约。

  停止：
  11. 点击【⏹ 停止】可终止整个签约过程。
      已完成的签约不会被撤销，仅跳过尚未处理的居民。

  签约完成：
  12. 所有居民处理完毕后（或手动停止后），日志区会显示汇总信息：
      "签约完成! 成功: X, 失败: Y, 总耗时: Z秒"

  13. 签约结束后可再次查询，确认签约结果是否已反映在3.0系统中。

────────────────────────────────────────
4.5 导出数据
────────────────────────────────────────

  任何时候都可以将当前表格中的居民数据导出为CSV文件：

  1. 点击表格上方的【导出列表】按钮。

  2. 在弹出的保存对话框中选择保存路径和文件名。
     默认文件名格式为"居民列表_日期_时间.csv"。

  3. 导出的CSV文件包含以下列：
     序号、姓名、身份证号、签约状态、签约团队、签约医生、
     签约日期、协议到期、PERSONID、合同编号

  4. 导出的文件可直接用 Excel 打开，中文显示正常。

  用途建议：
  • 签约前导出：保留原始居民名单以备核对
  • 签约后导出：留存签约结果作为工作记录

五、签约状态与工作流程详解
================================================================================

公卫3.0系统中的签约状态共有以下几种：

  状态          含义
  ─────────────────────────────────────────────────────
  未签约        该居民没有任何签约记录
  已签约        该居民已有有效的签约合同
  医生申请      医生端已发起签约，等待居民端确认
  居民申请      居民端已发起签约，等待医生端确认
  拒绝签约      签约申请被拒绝

本工具会根据居民当前状态自动选择最优操作（全自动判断，无需人工干预）：

  ┌─────────────────────────────────────────────────────────────┐
  │ 居民当前状态        工具自动执行的操作        最终结果       │
  ├─────────────────────────────────────────────────────────────┤
  │                                                             │
  │ 未签约          →  自动发起签约             → 医生申请      │
  │                     （从医生端申请）                         │
  │                                                             │
  │ 医生申请        →  已发起，标记为成功        → 保持不变     │
  │                     日志显示"待健康卡确认"                   │
  │                                                             │
  │ 居民申请        →  自动确认签约             → 已签约        │
  │                     （直接从医生端确认）                     │
  │                                                             │
  └─────────────────────────────────────────────────────────────┘

  详细说明：

  • 未签约的居民：工具会自动发起签约，使其变为"医生申请"状态。
    该状态下的合同需要通过健康卡终端完成最终确认。
    工具已完成它在医生端能做的全部操作。

  • 居民申请的居民：这些居民已通过健康卡端自行发起了签约申请，
    工具可以直接在医生端确认，一步到位变成"已签约"。
    这是效率最高的情况。

  • 医生申请的居民：之前已由医生端发起过签约。工具会将其标记为
    成功（因为发起步骤已完成），日志中显示"待健康卡确认"提示，
    表示仍需通过健康卡终端完成最终确认。

六、特殊场景操作指南
================================================================================

────────────────────────────────────────
6.1 查询并重签已过期协议的居民
────────────────────────────────────────

  部分居民的签约协议已到期，但在系统中仍显示为"已签约"（未解约），
  这些居民需要重新签约。

  操作步骤：
  1. 签约状态下拉框选择"已签约"
  2. 在"协议到期日(前)"栏填入今天的日期
     格式为纯数字，例如：20260404（表示2026年4月4日）
  3. 点击【查询全部】
  4. 表格中将显示所有协议到期日早于指定日期的已签约居民
     （即协议已过期的居民）
  5. 确认数据无误后，选中需要重签的居民
  6. 点击【▶ 开始签约】执行重新签约

────────────────────────────────────────
6.2 处理"医生申请"状态的居民
────────────────────────────────────────

  这些居民之前已由医生端发起过签约申请，但尚未通过健康卡端完成确认。

  查看方法：
  1. 签约状态下拉框选择"医生申请"
  2. 点击【查询全部】
  3. 即可查看所有处于此状态的居民列表

  处理说明：
  • 如果选中这些居民并执行签约，工具会将其标记为"已发起"
    （因为发起步骤确实已完成），日志中显示"待健康卡确认"
  • 这类居民的最终确认需要通过健康卡终端设备完成

────────────────────────────────────────
6.3 处理"居民申请"状态的居民
────────────────────────────────────────

  这些居民已通过健康卡端自行发起了签约申请，等待医生端确认。

  操作步骤：
  1. 签约状态下拉框选择"居民申请"
  2. 点击【查询全部】
  3. 选中需要确认的居民
  4. 点击【▶ 开始签约】
  5. 工具会自动在医生端确认这些申请，签约立即生效（变为"已签约"）

  这是效率最高的签约方式，只需一步确认即可完成。

────────────────────────────────────────
6.4 使用多个医生账号轮换操作
────────────────────────────────────────

  公卫3.0系统对每个医生ID有每日签约人次限制（通常为80人次/ID/天）。
  当一个账号的签约量接近或达到上限时，需要更换账号继续操作。

  操作步骤：
  1. 当前账号签约完成后（或达到80人上限时），无需关闭程序
  2. 直接在登录区的"账号"栏修改为新的医生账号
  3. 在"密码"栏输入新账号的密码
  4. 点击【登 录】
  5. 登录成功后，"签约医生"和"签约团队"会自动更新为新账号对应的信息
  6. 重新查询居民列表（建议重新查询以确保数据准确）
  7. 选中尚未签约的居民，继续批量签约

  提示：切换账号后，建议先查询确认上一个账号已签约的居民是否已
        变更状态，避免重复操作。

七、配置自动保存说明
================================================================================

程序会自动记住您的使用偏好，下次启动时自动恢复，减少重复输入。

  自动保存的内容：
  • 系统地址
  • 登录账号（密码不会保存，每次需重新输入）
  • 机构代码
  • 签约医生姓名
  • 签约团队名称
  • 间隔时间设置

  保存时机：
  • 每次登录成功后自动保存
  • 每次开始签约时自动保存
  • 关闭程序时自动保存

  配置文件：
  • 保存在 GulfSign.exe 同目录下的 gulfsign_config.json 文件中
  • 如需重置所有配置，删除该文件即可，下次启动将恢复默认设置


================================================================================
八、常见问题与解决方案
================================================================================

问：首次双击 GulfSign.exe 后弹出安全警告怎么办？
答：点击"更多信息"，然后点击"仍要运行"。这是 Windows 对未签名程序
    的正常提示，程序本身是安全的。

问：杀毒软件拦截了 GulfSign.exe 怎么办？
答：将 GulfSign.exe 添加到杀毒软件的信任列表（白名单）中，然后
    重新运行。常见杀毒软件的操作方式：
    • Windows Defender：打开"病毒和威胁防护"→"允许的威胁"中添加
    • 360安全卫士：在弹出的拦截提示中选择"信任"或"允许"

问：点击登录后提示"请填写完整的登录信息"
答：请确保系统地址、账号、密码三个栏位都已填写，不能留空。

问：登录失败，提示"请求参数错误"
答：账号或密码不正确。请核对后重试。建议先在浏览器中用同样的账号
    密码登录3.0系统网页确认是否能正常登录。

问：登录失败，提示网络相关错误
答：请检查：
    (1) 电脑网络连接是否正常
    (2) 能否在浏览器中打开 https://ggws.hnhfpc.gov.cn
    (3) 如果3.0系统本身也打不开，说明服务器正在维护，请稍后重试

问：查询结果为0条记录
答：可能原因：
    (1) 查询条件过于严格 — 试着放宽条件（如将状态改为"全部"）
    (2) 机构代码不正确 — 尝试清空机构代码后重新查询
    (3) 该机构下确实没有符合条件的居民

问：签约时部分居民提示"操作失败"
答：可能原因：
    (1) 该居民的基础信息不完整，3.0系统拒绝签约
    (2) 服务器暂时繁忙，可暂停后稍等片刻再继续
    (3) 当前医生ID的签约人次已达上限（通常80人次/天），
        需要更换账号（参见 6.4 节）

问：日志中显示"待健康卡确认"是什么意思？
答：这表示工具已经从医生端成功发起了签约申请，但该类型的签约需要
    通过健康卡终端设备完成最终确认。这不是错误，而是正常的业务流程。
    工具已经完成了它在医生端能做的全部操作。

问：签约速度能达到多少？
答：取决于3.0系统服务器的响应速度和您的网络环境。
    一般情况下，间隔设置为0.3~0.5秒时，实际签约速度约为1.5~3秒/人。
    网络良好时每小时可处理约1200~2400人。

问：可以同时打开多个程序窗口操作不同账号吗？
答：可以，但不建议用同一个账号同时操作（会导致会话冲突）。
    如果使用不同的医生账号，可以分别打开多个窗口同时操作。

问：导出的CSV文件用Excel打开后中文显示乱码
答：正常情况下不会出现乱码。如果遇到此问题：
    (1) 不要直接双击打开，改为在 Excel 中选择"数据"菜单
    (2) 点击"从文本/CSV导入"
    (3) 在编码选项中选择"UTF-8"
    (4) 点击"加载"即可正常显示

问：关闭程序后再次打开，之前的查询结果还在吗？
答：不在。程序只保存配置信息（账号、医生名等），查询结果和日志
    在关闭后不会保留。如需保留查询结果，请在关闭前使用"导出列表"
    功能保存为CSV文件。


================================================================================
九、注意事项与最佳实践
================================================================================

  1. 首次使用建议先选中少量居民（如3~5人）测试签约流程，
     确认一切正常后再进行大批量签约。

  2. 签约前务必先用【查询(首页)】预览数据，确认查询条件正确后，
     再用【查询全部】加载全部数据。

  3. 间隔时间建议保持默认的 0.5 秒，或设置为 0.3~1.0 秒之间。
     不建议设置为 0，过快的请求频率可能被服务器限流。

  4. 签约过程中如需中断，请点击【⏹ 停止】按钮，而不是直接关闭窗口。
     如果直接关闭窗口，程序会弹出确认对话框请您确认。

  5. 每个医生ID每日签约上限通常为80人次。建议提前规划好需要几个
     医生账号来完成当天的签约任务。接近上限时及时更换账号。

  6. 签约完成后建议立即导出一份CSV文件备份，记录本次操作的结果，
     以备日后核对。

  7. 日志区记录了每次操作的详细结果和时间。如遇到问题需要反馈，
     可截图日志区的内容。点击【清空日志】可清除历史日志。

  8. 每次启动程序后需重新输入密码（密码不会被保存），
     其他配置项（账号、医生、团队等）会自动恢复。

  9. 如遇3.0系统维护或不稳定（频繁超时、报错），建议暂停操作，
     待系统恢复正常后再继续。
