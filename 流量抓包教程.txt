================================================================================
                        「流量抓包」标签页 — 使用教程
================================================================================

一、这个工具是做什么的
--------------------------------------------------------------------------------
在软件中打开「流量抓包」标签页，点击「开始抓包」后，程序会：

  • 在本机启动代理（默认端口 8888）
  • 尝试安装本机信任的 CA 证书（用于解密 HTTPS）
  • 自动把 Windows / macOS 的系统 HTTP/HTTPS 代理指向本机 127.0.0.1:8888

随后，符合「抓包名单」的 HTTPS 请求与响应会被写入程序目录下的
traffic_log.txt（界面里也会显示「已记录请求」条数与日志大小）。操作结束后可点
「导出日志」把文件另存到指定位置，交给技术人员分析。

重要说明（请务必阅读）：

  1）当前版本只会对「名单内的域名」做解密并记录「完整」请求与响应正文。
     名单外的网站仍会正常联网，但不会被记录详细内容。

  2）抓包日志里可能出现 Token、OpenID、业务参数、身份证号片段等敏感信息。
     请仅在自己有权操作的设备、本人或测试账号上使用；导出、传输、保存
     时注意保密，必要时可先自行脱敏后再发送。

  3）抓包时不要与本软件「获取OpenID」页的代理同时开启，需先停其中一个。

  4）本软件内「3.0系统签约」「健康卡确认」等页面的联网已尽量不走系统代理，
     但若其他软件依赖系统代理，抓包期间其行为可能受影响；分析完成后务必点
     「停止抓包」以清除系统代理。

  5）界面日志若出现「已记录 [某域名] …」表示：该 HTTPS 请求被代理解密并写入
     traffic_log.txt，同时代理会把请求正常转发给服务器、把响应返回给微信。
     这不是「被拦下不能下一步」；若小程序里真的卡住，多半是证书未信任或网络
     问题，见下文「九、常见问题」。


二、当前会「完整记录」的域名（抓包名单）
--------------------------------------------------------------------------------
下列域名走本机解密并写入 traffic_log.txt（与「获取OpenID」共用同一套规则）：

  • jkkyljl.hnhfpc.gov.cn      — 湖南省健康卡医疗机构联网平台（签约确认等）
  • jkkgzh.hnhfpc.gov.cn       — 湖南省居民健康卡（绑卡/解绑等 H5）
  • jkkzc.hnhfpc.gov.cn        — 湖南健康卡相关站点
  • health.tengmed.com         — 腾讯健康相关
  • h5-health.tengmed.com      — 腾讯健康 H5
  • wechat.wecity.qq.com       — 微信侧健康/城市服务相关
  • card.wecity.qq.com         — 健康卡/微卡相关

不在上述名单内的地址（例如浏览器直接打开 ggws.hnhfpc.gov.cn 公卫 3.0 网页、
普通新闻网站等），当前版本不会把其 HTTPS 内容写入本抓包日志。


三、推荐环境：电脑端微信 + 小程序（最常用）
--------------------------------------------------------------------------------
适用：需要在「湖南省居民健康卡」「我的健康卡」等小程序里完成绑卡、解绑、
      人脸验证、家医相关页面操作，并把接口流程记录下来。

操作步骤：

  1. 关闭「获取OpenID」页的代理（若曾开启）。
  2. 打开「流量抓包」→ 点击「开始抓包」。
  3. 若弹出证书安装提示，请按提示完成安装（Windows 常见为导入「受信任的根证书」；
     需管理员权限时请允许）。
  4. 完全退出电脑版微信后重新打开微信（确保新开的进程走系统代理）。
  5. 在电脑微信里打开目标小程序，按下面「五、建议操作清单」逐项操作一遍。
  6. 观察界面下方「已记录请求」数字是否增加、日志大小是否变大。
  7. 点击「停止抓包」→「导出日志」，保存 txt 并发给技术人员。

若小程序页面一直空白或无法加载：

  • 确认已安装 CA 且系统代理为 127.0.0.1:8888。
  • 确认本软件抓包页显示「抓包运行中」。
  • 仍不行可尝试重启微信；部分环境需暂时关闭第三方安全软件对代理的拦截。


四、可选：手机连同一 WiFi 抓小程序（补充场景）
--------------------------------------------------------------------------------
若必须在手机微信小程序里操作（电脑版没有某功能或行为不一致）：

  1. 电脑运行本软件，「流量抓包」中「开始抓包」（代理监听 8888）。
  2. 在手机 WiFi 设置里，将 HTTP 代理设为「手动」：
     服务器填电脑在局域网中的 IP（与「获取OpenID」教程中一致），端口 8888。
  3. 手机浏览器访问：http://电脑IP:8888/cert 下载并安装 CA 证书；
     iOS 需在「设置 → 通用 → 关于本机 → 证书信任设置」中对该 CA 开启「完全信任」。
  4. 打开手机微信，进入同一小程序，完成操作。
  5. 回到电脑「停止抓包」并「导出日志」。

注意：部分 Android 版本上，微信可能不信任用户安装的证书，导致无法解密，
      此时以电脑微信抓包为主更稳妥。


五、建议操作清单（尽量做全，便于分析绑卡 / 解绑 / 人脸 / 家医）
--------------------------------------------------------------------------------
在「开始抓包」之后、在微信或浏览器中按顺序操作（可重复多轮）：

【居民健康卡 / 健康卡 H5】
  • 打开小程序首页，下拉刷新。
  • 进入「健康卡」列表：查看已绑卡、切换卡片（若有）。
  • 走一遍「添加 / 绑定健康卡」完整流程（含需人脸的步骤，如适用）。
  • 走一遍「解绑」或「管理卡片」里与解绑相关的步骤（注意平台 7 天限制等提示）。
  • 打开「电子健康卡」「二维码」等会拉取个人信息的页面。
  • 若有「家庭医生签约」「家医服务」等入口，进入并浏览会加载列表或详情的页面。

【与家医 / 签约相关的入口】
  • 从健康卡或卫健委相关入口进入任何指向 hnhfpc.gov.cn 或腾讯健康域名的页面，
    尽量多点几层（列表、详情、提交、返回）。

【可选：浏览器】
  • 在已设置系统代理的电脑上，用 Chrome / Edge 打开：
    https://jkkgzh.hnhfpc.gov.cn/hunan/#/
    登录后操作与小程序类似的绑卡、查看等（仅当业务支持网页版时）。

每完成一类操作，可看界面「已记录请求」是否增加；若长时间为 0，说明流量未经过代理
或未命中上述域名，需检查代理、证书或是否用了未走系统代理的客户端。


六、关于「是否能看到账号密码、登录凭证」
--------------------------------------------------------------------------------
  • 现代应用多数使用微信登录或 Token，日志里更常见的是 Token、Cookie、
    openId、业务 JSON，而不是明文密码。
  • 若某接口以表单或 JSON 明文提交账号密码，理论上会出现在 traffic_log.txt 中。
  • 因此强烈建议：仅使用本人或测试账号；抓包结束后妥善保管日志；发送前可
    删除与任务无关的片段。
  • 请勿在未经授权的情况下对他人账号或设备进行抓包。


七、界面按钮说明
--------------------------------------------------------------------------------
  • 开始抓包：启动代理、安装证书、设置系统代理。
  • 停止抓包：停止代理、清除系统代理（建议每次用完都点）。
  • 导出日志：把当前 traffic_log.txt 复制到您选择的路径（带时间戳文件名）。
  • 清空日志：只清空窗口里的文字显示，不删除已写入磁盘的 traffic_log.txt；
    新一次「开始抓包」通常会重新记录（以程序实际行为为准）。
  • 已记录请求 / 日志大小：反映名单内域名被记录的次数及文件体积；体积过小可能
    表示未命中目标流量。


八、抓包结束后必做
--------------------------------------------------------------------------------
  1. 点击「停止抓包」，确认系统代理已清除（避免影响日常上网或其它软件）。
  2. 使用「导出日志」保存一份副本。
  3. 若随后要在本软件登录公卫 3.0 或调用健康卡接口，在代理已关闭的前提下
     一般可正常使用；若遇连接异常，先确认系统代理未残留。


九、常见问题
--------------------------------------------------------------------------------
  Q：已记录数一直是 0？
  A：检查是否「开始抓包」、证书是否安装成功、微信是否重启、操作的是否为名单内
     业务（居民健康卡 / 相关 H5）。

  Q：日志里很多「已记录」，是不是把请求拦住了、所以不能点下一步？
  A：不是。已记录 = 边记日志边正常转发，小程序应能继续操作。若界面一直加载，
     请确认 CA 已装入「受信任根证书」、完全退出微信后再开、必要时暂时关闭
     杀毒/安全软件对本地代理的干扰。

  Q：想和 OpenID 一起抓怎么办？
  A：不要同时开两个标签的代理。可先「流量抓包」一轮导出，再开「获取OpenID」，
     或反之。

  Q：日志太大打不开？
  A：可用记事本类软件分批打开，或只截取与绑卡、解绑时间点相关段落发给技术人员，
     并说明操作时间线。
